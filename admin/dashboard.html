Rapports Google Analytics

Configurer Google Analytics

Pour afficher vos statistiques, vous devez connecter votre compte Google Analytics via Looker Studio.

1

Cloner le modèle Looker Studio

Utilisez notre modèle pré-configuré comme base.

Ouvrir le modèle
2

Connecter vos données

Cliquez sur les 3 points en haut à droite > "Créer une copie" > Sélectionnez votre source de données Google Analytics.

3

Obtenir le lien d'intégration

Fichier > Intégrer le rapport > Activer l'intégration > Copier l'URL d'intégration.

4

Configurer la variable

Ajoutez la variable ANALYTICS_EMBED_URL dans Cloudflare Pages avec votre lien.

Paramètres du Site

Configuration via Variables d'Environnement

La configuration de ce site est gérée directement via les variables d'environnement de votre hébergeur (ex: Cloudflare Dashboard ou wrangler.toml).

Les valeurs ci-dessous sont affichées à titre informatif et ne peuvent pas être modifiées ici.

Variable: BLOG_FEED_URL (Substack, WordPress, Ghost, Medium...)

Variable: YOUTUBE_FEED_URL

Variable: PODCAST_FEED_URL

Variable: WSTD_STAGING_URL (Optionnel - Active le proxy vers Webstudio)

Foire Aux Questions

Qu'est-ce que iziWebCMS ?
iziWebCMS est un CMS Headless moderne qui connecte vos sources de données existantes (Substack, YouTube, Podcast) à un frontend ultra-rapide. Il vous permet de gérer votre contenu centralisé sans base de données complexe.
Comment mettre à jour mon contenu ?
C'est automatique ! iziWebCMS se synchronise avec vos flux RSS (Substack, YouTube, Anchor). Dès que vous publiez sur ces plateformes, votre site est mis à jour après expiration du cache (environ 3 minutes). Vous pouvez forcer la mise à jour avec le bouton "Vider le Cache".
Comment changer le nom de mon site ?
Le nom de votre site est défini par la variable d'environnement META_TITLE dans Cloudflare Pages. Vous pouvez le modifier dans les paramètres de votre projet Cloudflare > Settings > Environment Variables.
Le Frontend Builder ne fonctionne pas
Assurez-vous d'avoir configuré la variable FRONTEND_BUILDER_URL avec le lien vers votre projet Webstudio. Si ce lien est manquant, le bouton d'accès au constructeur sera désactivé.
Pourquoi mes images ne s'affichent pas ?
iziWebCMS utilise un proxy intelligent pour afficher votre site Webstudio sur votre propre domaine. Si des images manquent, c'est souvent lié à une configuration DNS ou CSP. Le système gère, cependant, automatiquement la plupart des cas via le proxy intégré.
